commit message format:
`<type>: <subject>`

type:(!must!)
+ feat: 新功能
+ fix: 修复
+ docs: 文档
+ style: 格式（不影响代码运行的变动）
+ refactor: 重构（即不是新增功能，也不是修改bug的代码变动）
+ test: 测试
+ chore: 构建过程或辅助工具的变动
+ revert: 撤销之前的commit
+ merge: 合并分支
+ sync: 同步分支


subject:(!must!) 描述
